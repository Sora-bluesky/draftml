# 要件定義書

> draftml — AutoCAD → FreeCAD + Claude Code MCP 移行プロジェクト

**最終更新**: 2026-03-23
**ステータス**: 承認済み（Phase 0 調査完了時点）

---

## 1. プロジェクト概要

| 項目           | 内容                                                                                                     |
| -------------- | -------------------------------------------------------------------------------------------------------- |
| プロジェクト名 | draftml                                                                                                  |
| 目的           | AutoCAD から FreeCAD + Claude Code MCP への移行による、コスト削減と図面作図の自動化                      |
| 対象業務       | 建材の製造図面作図（2D製造図・断面図・寸法記入・PDF/DXF出力）                                            |
| 背景           | AutoCAD のライセンス費（年間数百万円規模）が経営上の課題であり、オープンソースCAD + AI自動化で解決を図る |

---

## 2. ビジネス要件

| ID   | 要件                           | 定量目標                                       | 優先度 |
| ---- | ------------------------------ | ---------------------------------------------- | ------ |
| BR-1 | AutoCAD ライセンス費の完全廃止 | 年間数百万円規模の削減                         | 必須   |
| BR-2 | 製造図面作図の自動化           | 作図工数の60〜80%削減                          | 必須   |
| BR-3 | 既存DWGファイル資産の継続利用  | 主要DWGファイルをFreeCADで閲覧・編集可能にする | 必須   |

---

## 3. 機能要件

### 3.1 図面作成

| ID   | 要件                   | 詳細                                                                               |
| ---- | ---------------------- | ---------------------------------------------------------------------------------- |
| FR-1 | 2D製造図の作成         | 正面図・側面図・断面図を含む製造図面を生成できること                               |
| FR-2 | 寸法の自動記入         | 3Dモデルから2D図面を生成する際に、主要寸法が自動で記入されること                   |
| FR-3 | 図枠テンプレートの適用 | A3縦の社内標準図枠（会社名・品番・図面番号・作成者・日付・改訂欄）を適用できること |
| FR-4 | 複数形式での出力       | PDF（主出力）、DXF（ジオメトリデータ）、SVG で図面を出力できること                 |

### 3.2 一括処理

| ID   | 要件                               | 詳細                                                                    |
| ---- | ---------------------------------- | ----------------------------------------------------------------------- |
| FR-5 | CSVからの一括図面生成              | 品番リスト（CSV形式）を読み込み、各品番の図面を自動で一括生成できること |
| FR-6 | パラメトリックモデルによる寸法変更 | Spreadsheet上のパラメータ変更で3Dモデルと図面が連動して更新されること   |

### 3.3 データ変換

| ID   | 要件                       | 詳細                                                         |
| ---- | -------------------------- | ------------------------------------------------------------ |
| FR-7 | 既存DWGファイルのDXF変換   | AutoCAD形式（DWG）のファイルをDXF形式に一括変換できること    |
| FR-8 | 変換後ファイルの閲覧・編集 | 変換されたDXFファイルをFreeCADで開き、閲覧・編集ができること |

---

## 4. 非機能要件

| ID    | 要件         | 詳細                                                                      |
| ----- | ------------ | ------------------------------------------------------------------------- |
| NFR-1 | 個人運用     | 専任のCADオペレーターでなくても運用できること（非エンジニアでも操作可能） |
| NFR-2 | 自然言語操作 | Claude Code MCP を通じて、自然言語の指示で図面の作成・編集ができること    |
| NFR-3 | 製造現場品質 | 生成された図面が製造現場で使用できる品質（寸法精度・視認性）を満たすこと  |
| NFR-4 | 再現性       | 同じパラメータからは常に同じ図面が生成されること                          |

## 7. 用語集

| 用語                 | 説明                                                                               |
| -------------------- | ---------------------------------------------------------------------------------- |
| 製造図面             | 製品を製造するために必要な寸法・形状・材質等を記載した図面                         |
| 3面図                | 正面図・側面図・上面図（平面図）の3方向から見た投影図                              |
| 断面図               | 製品の内部構造を示すために切断面を表示した図                                       |
| 図枠                 | 図面の外枠。表題欄（品番・作成者等の情報欄）を含む                                 |
| DWG                  | AutoCAD のネイティブファイル形式                                                   |
| DXF                  | Drawing Exchange Format。CADソフト間のデータ交換に使われるオープン形式             |
| TechDraw             | FreeCAD の2D図面作成ワークベンチ                                                   |
| PartDesign           | FreeCAD の3Dモデリングワークベンチ                                                 |
| MCP                  | Model Context Protocol。Claude Code と外部ツールを接続するプロトコル               |
| パラメトリックモデル | 寸法をパラメータ（変数）で定義し、値の変更でモデル全体が連動して更新される設計手法 |
| C型鋼                | 断面がC字型の鋼材。建材として広く使われる                                          |
| PoC                  | Proof of Concept（概念実証）。本番導入前に実現可能性を検証すること                 |
