# ADR-0001: CADソフトウェアの選定

- **日付**: 2026-03-23
- **ステータス**: accepted
- **決定者**: AI変革推進担当

## 背景

現在、建材の製造図面作図に AutoCAD を使用しているが、ライセンス費が年間数百万円規模かかっている。コスト削減と作図業務の自動化を両立するため、代替CADソフトウェアの選定が必要となった。

選定にあたっての判断基準:

1. **コスト**: ライセンス費の大幅削減（理想はゼロ）
2. **機能**: 2D製造図面（正面図・断面図・寸法記入）の作成が可能であること
3. **自動化**: Claude Code MCP との連携による自然言語操作が可能であること
4. **互換性**: 既存DWGファイル資産を活用できること
5. **出力形式**: PDF・DXF・SVG での出力に対応していること

## 決定内容

**FreeCAD 1.0** を採用する。

## 検討した代替案

### 1. AutoCAD（現行）の継続

- メリット: 移行リスクなし、DWG完全互換、豊富なエコシステム
- デメリット: **年間数百万円規模のライセンス費**、MCP連携なし（Claude Code による自動化が困難）

### 2. FreeCAD 1.0（採用）

- メリット: **無料（オープンソース）**、MCP対応（contextform/freecad-mcp が存在）、Python API による自動化、PartDesign + TechDraw で製造図面作成可能
- デメリット: DWG直接編集は不可（DXF変換が必要）、TechDraw の成熟度が AutoCAD に劣る

### 3. LibreCAD

- メリット: 無料、軽量な2D CAD
- デメリット: 3Dモデリング不可、MCP連携なし、パラメトリック設計の制限

### 4. BricsCAD / DraftSight

- メリット: DWG完全互換、AutoCAD からの移行が容易
- デメリット: 有償ライセンス（コスト削減効果が限定的）、MCP連携なし

## 選定理由

1. **コスト**: 完全無料であり、BR-1（年間数百万円規模の削減）を最大限に達成できる
2. **MCP連携**: `contextform/freecad-mcp` が存在し、Claude Code からの自然言語操作が可能（BR-2 の実現手段）
3. **パラメトリック設計**: Spreadsheet ワークベンチによるパラメータ駆動設計が可能で、CSV一括生成（FR-5）の基盤となる
4. **Python API**: FreeCAD 内蔵の Python インタプリタにより、スクリプトベースの自動化が可能
5. **オープンソース**: ベンダーロックインがなく、長期的な持続可能性が高い

## リスクと軽減策

| リスク                                        | 影響度 | 軽減策                                                                                       |
| --------------------------------------------- | ------ | -------------------------------------------------------------------------------------------- |
| TechDraw の図面品質が製造現場基準を満たさない | 高     | Phase 1 PoC（B-2）で実製品図面を検証し、品質が不足する場合は手動調整のワークフローを確立する |
| DWG → DXF 変換時にジオメトリが崩れる          | 中     | ODA File Converter で変換後に目視確認。崩れた場合は Claude Code に3D再モデリングを指示する   |
| FreeCAD のバージョンアップによる互換性問題    | 低     | バージョン固定運用。更新前に検証環境で動作確認を行う                                         |

## 結果・影響

- すべての図面作成ワークフローが FreeCAD ベースに移行する
- AutoCAD ライセンスは Phase 2（3〜6ヶ月後）に更新停止
- 既存 DWG ファイルは ODA File Converter で DXF に変換して利用する
- 自動化パイプラインは FreeCAD の Python API + MCP を基盤として構築する
