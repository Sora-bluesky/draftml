# ADR-0003: 図面生成方式の選定

- **日付**: 2026-03-24
- **ステータス**: accepted
- **決定者**: AI変革推進担当
- **前提ADR**: ADR-0001（FreeCAD採用）、ADR-0002（MCP選定）

## 背景

Phase 1 で FreeCAD + MCP パイプラインの実証が完了した（C型鋼PoC）。Phase 2 では MDF木質建材（ドア枠・窓枠等、100品番以上）の製造図面を自動生成する。

M6 Discovery の分析により、以下の事実が判明した:

1. **3Dモデルは不要**: 製造図面は2D断面図・穴位置図・姿図で構成される（`docs/product-analysis.md` §1-5）
2. **既存DWGは第三角法に則っていない**: ビュー配置がJIS B 0001非準拠のため、テンプレートとしてそのまま使えない
3. **96%の寸法が固定**: 334寸法中322個は高さバリエーションで変化しない。パラメトリック寸法は12個のみ（§2）
4. **12シート構成**: 1品番あたり12シート・13パーツ。各シートはモルダー断面(1:1)、姿図(1:10)、穴詳細図で構成（§5）

これらを踏まえ、図面生成方式の技術選定が必要となった。

### 判断基準

| 優先度 | 基準                       | 根拠                                              |
| ------ | -------------------------- | ------------------------------------------------- |
| 1      | 第三角法（JIS B 0001）準拠 | 製造現場の品質要件。既存DWG非準拠のため必須       |
| 2      | バッチ処理効率             | 100品番以上 × 12シート = 1,200シート以上の生成    |
| 3      | 既存図面資産の活用         | 断面プロファイル等（96%固定部分）の再利用         |
| 4      | 保守性                     | パーツ追加・寸法変更への対応容易性                |
| 5      | FreeCAD依存度の最適化      | ADR-0001のFreeCAD採用を活かしつつ過度に依存しない |

## 決定内容

**ハイブリッド方式（方針D）** を採用する。第三角法準拠マスターテンプレートの初回作成にはFreeCAD（2D Draft + TechDraw）を使用し、量産バッチ処理にはezdxf（Pythonライブラリ）でパラメトリック寸法書換えを行う。

## 検討した代替案

### 方針A: FreeCAD TechDraw（Phase 1方式の拡張）

3Dモデルを作成し、TechDraw で2D投影図を生成する。Phase 1（C型鋼）で検証済みの方式。

- メリット:
  - Phase 1で実証済み。プロンプトテンプレート・品質チェック手順が流用可能
  - TechDrawの寸法ツールが充実（公差表示、引出線、バルーン）
  - MCP連携（execute_python）で自動化済み
- デメリット:
  - **3Dモデルが必要**（ユーザー方針「3D不要」と矛盾）
  - MDF断面プロファイル（R面取り・溝・ラッピング範囲）の3Dモデリングが複雑
  - 12シート × 13パーツの3Dモデル管理が煩雑
  - FreeCAD GUIセッション必須 → バッチ処理のボトルネック

**判定**: ユーザー方針「3Dモデル不要」により **不採用**。

### 方針B: FreeCAD 2D Draft（2D直接生成）

FreeCAD の Draft ワークベンチで2D図面を直接作成する。3Dモデルは不要。

- メリット:
  - 3Dモデル不要（ユーザー方針に合致）
  - FreeCADの視覚的環境でテンプレート設計可能
  - execute_python でスクリプト化可能
- デメリット:
  - Draft ワークベンチの寸法ツールはTechDrawより限定的
  - **FreeCAD GUIセッション必須** → バッチ処理が低速
  - 12シート × 100品番以上の生成にFreeCADセッション管理が必要
  - DXF出力の品質がezdxf直接生成に劣る場合がある

**判定**: テンプレート設計フェーズでは有用だが、量産には不向き。

### 方針C: ezdxf（DXFテンプレート操作）

Pythonライブラリ ezdxf で DXF ファイルを直接操作する。テンプレートDXFの寸法値をパラメトリックに書換えてバッチ出力。

- メリット:
  - **ヘッドレス実行**（GUIセッション不要、サーバ実行可）
  - 純Python → バッチ処理が高速・並列化容易
  - DXF直接操作 → AutoCAD完全互換の出力品質
  - 96%固定・4%パラメトリックの構造にテンプレート方式が最適
- デメリット:
  - **新規テンプレートを最初から作成する必要**（既存DWGは第三角法非準拠）
  - テンプレート作成時に視覚的フィードバックがない（コード→出力→確認のサイクル）
  - ezdxf は新規パッケージ依存（pip install ezdxf）
  - PDF出力には追加ツールが必要（ODA Converter または matplotlib）

**判定**: 量産フェーズでは最適だが、初回テンプレート設計に視覚環境がないのが弱点。

### 方針D: ハイブリッド（採用）

2フェーズに分離する:

- **設計フェーズ**（一度きり）: FreeCAD で第三角法準拠マスターテンプレートを作成 + 既存DXFから断面プロファイル等を抽出・再配置
- **量産フェーズ**（継続）: ezdxf でマスターテンプレートのパラメトリック寸法を書換え、バッチ出力

- メリット:
  - 設計フェーズ: FreeCADの視覚環境でテンプレートの正確性を保証
  - 量産フェーズ: ezdxfのヘッドレス処理でバッチ効率を最大化
  - **既存図面資産を活用**: 断面プロファイル（96%固定部分）を既存DXFから抽出・再利用
  - 関心の分離: 設計（正確性重視）と量産（効率重視）で最適なツールを使い分け
- デメリット:
  - 2つのツールチェーンの習得・保守が必要
  - テンプレートのDXFエンティティ構造を設計・量産で共有するための規約が必要

## 選定理由

1. **ユーザー方針との整合**: 「3Dモデル不要」に完全準拠。方針Aを明確に排除できる
2. **既存資産の活用**: 既存DXFの断面プロファイル（322/334寸法 = 96%が固定）はそのまま再配置で活用可能。全面再描画を回避
3. **バッチ効率**: 100品番 × 12シート = 1,200枚以上をezdxfヘッドレス処理で生成。FreeCADセッション管理のボトルネックを排除
4. **品質保証**: マスターテンプレートをFreeCADで視覚的に検証した上で量産に移行。「一度正しく作り、何度も使う」パターン
5. **AutoCAD互換**: ezdxfのDXF出力はAutoCAD R2018形式に準拠。製造現場のAutoCAD環境との互換性を維持
6. **段階的移行**: Phase 1のFreeCAD + MCP投資を無駄にせず、Phase 2の要件（2D製造図面の大量生成）に最適化

### 2フェーズの詳細

#### 設計フェーズ（マスターテンプレート作成）

```
既存DXF（第三角法非準拠）
    │
    ├─ ezdxf で断面プロファイル・寸法エンティティを抽出
    │
    ▼
FreeCAD Draft で第三角法準拠レイアウトに再配置
    │
    ├─ JIS B 0001 ビュー配置ルール適用
    ├─ 表題欄テンプレート（社内規格準拠）
    ├─ 投影法シンボル（第三角法）
    ├─ 公差注記（JIS B 0405-c / m級）
    │
    ▼
マスターテンプレートDXF（12シート分）
    │
    └─ パラメトリック寸法にマーカー付与（ezdxfで検索・置換可能な構造）
```

#### 量産フェーズ（バッチ生成）

```
Excel / CSV（入力パラメータ）
    │
    ├─ frame_height: 枠全高
    ├─ opening_height: 開口高さ
    │
    ▼
ezdxf パイプライン（ヘッドレスPython）
    │
    ├─ マスターテンプレートDXF をロード
    ├─ パラメトリック寸法12個を書換え（全高連動10個 + 開口高連動2個）
    ├─ 品番テキスト・表題欄を更新
    ├─ 品質チェック（寸法整合性・エンティティ数・BoundBox）
    │
    ▼
出力: {品番}_sheet{N}.dxf + {品番}_sheet{N}.pdf（12シート × N品番）
```

## リスクと軽減策

| リスク                                                                                         | 影響度 | 軽減策                                                                                                                           |
| ---------------------------------------------------------------------------------------------- | ------ | -------------------------------------------------------------------------------------------------------------------------------- |
| ezdxf でのDXFエンティティ操作が既存図面の複雑な構造（ブロック・ハッチ・MTEXT等）に対応できない | 高     | Step 5プロトタイプ（パーツ1B 1シート分）で早期検証。ezdxf は DXF R2018 を完全サポートしており、MTEXT・HATCH・INSERT 全て操作可能 |
| マスターテンプレートのDXF構造が量産フェーズのezdxf処理と整合しない                             | 中     | テンプレート設計時にパラメトリック寸法マーカーの命名規約を策定。テンプレート検証スクリプトを作成                                 |
| PDF出力品質が製造現場基準を満たさない                                                          | 中     | ODA File Converter（インストール済み）で DXF → PDF 変換。変換品質をプロトタイプで検証                                            |
| ezdxf の新規パッケージ依存                                                                     | 低     | ezdxf は Python の主要DXFライブラリ（PyPI 月間DL 100万+、10年以上の開発歴）。単一依存で済む                                      |
| FreeCAD の Draft ワークベンチでの第三角法レイアウト作成が想定より困難                          | 低     | 代替案: ezdxf で直接テンプレートを構築し、FreeCAD は検証・閲覧のみに使用                                                         |

## 結果・影響

### アーキテクチャへの影響

- **パイプライン構成の変更**: Phase 1 の `FreeCAD MCP → TechDraw → PDF/DXF` から、Phase 2 は `ezdxf → DXF → ODA → PDF` に主軸を移行
- **FreeCADの役割変更**: 図面生成エンジンからテンプレート設計・検証ツールへ。MCP連携は引き続き有効（テンプレート設計フェーズで使用）
- **新規依存**: `ezdxf`（pip install ezdxf）。Phase 2 のコアライブラリとして追加

### ドキュメントへの影響

- `docs/03-detailed-design.md`: パイプライン設計を2フェーズ構成に更新
- `docs/product-analysis.md` §8: 入力パラメータスキーマはそのまま適用

### Phase 1 資産の継続活用

| Phase 1 資産                | Phase 2 での活用                                   |
| --------------------------- | -------------------------------------------------- |
| FreeCAD 1.0 インストール    | テンプレート設計・DXF検証・閲覧                    |
| contextform/freecad-mcp     | テンプレート設計フェーズでのMCP操作                |
| `scripts/batch_generate.py` | 構造を参考にezdxf版を新規作成                      |
| A3図枠テンプレートSVG       | FreeCAD TechDraw用。ezdxf版は DXF 図枠として再作成 |
| execute_python 優先原則     | テンプレート設計フェーズで引き続き適用             |
| ODA File Converter          | DXF → PDF 変換に使用                               |
