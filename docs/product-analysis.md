# 製品分析レポート: MDF製ドア枠

> draftml Phase 2 — M6 Discovery 成果物

**分析日**: 2026-03-24
**対象**: IS-S-KMD-CS-R-900-C1 / IS-S-KMD-CS-R-1300-C1（ドア枠、高さ違い）

---

## 1. 分析概要

| 項目               | 900mm版   | 1300mm版 | 比較               |
| ------------------ | --------- | -------- | ------------------ |
| エンティティ総数   | 1,344     | 1,344    | **完全一致**       |
| 寸法数             | 334       | 334      | **完全一致**       |
| パーツ数           | 13        | 13       | **完全一致**       |
| シート数           | 12        | 12       | **完全一致**       |
| 固定寸法           | 322 (96%) | —        | 断面プロファイル等 |
| パラメトリック寸法 | 12 (3%)   | —        | 高さ連動           |

**結論**: 2つの図面は構造が完全に同一で、高さに連動する12個の寸法のみが変化する。パラメトリック生成に最適。

## 2. パラメトリック寸法の変化法則

### 法則1: 全高連動（+400mm）— 10個

| 寸法値(900) | 寸法値(1300) | Delta | 推定対象             |
| ----------- | ------------ | ----- | -------------------- |
| 901.0       | 1301.0       | +400  | 枠材全高（外寸）     |
| 899.0       | 1299.0       | +400  | 枠材全高（内寸）     |
| 895.0       | 1295.0       | +400  | 枠材全高（加工寸法） |
| 951.0       | 1351.0       | +400  | 縦枠長さ             |
| 954.0       | 1354.0       | +400  | 縦枠長さ（別パーツ） |

### 法則2: 開口高連動（+250mm）— 2個

| 寸法値(900) | 寸法値(1300) | Delta | 推定対象             |
| ----------- | ------------ | ----- | -------------------- |
| 539.0       | 789.0        | +250  | 開口高さ（内法）     |
| 509.8       | 759.8        | +250  | 開口高さ（加工基準） |

### パラメータ関係の推定

```
全高 = 基準値（900 or 1300）
枠外寸 = 全高 + 1
枠内寸 = 全高 - 1
加工寸法 = 全高 - 5
縦枠A = 全高 + 51
縦枠B = 全高 + 54
開口高 = 全高 × 係数 or 独立パラメータ
```

> 開口高の変化（+250mm）が全高の変化（+400mm）と異なるため、開口高は独立パラメータの可能性がある。追加品番での検証が必要。

## 9. 第三角法への修正要件

既存DWGは第三角法（JIS B 0001）に則っていない。Phase 2 で生成する図面は以下に準拠すること:

| 要件           | 規格                | 対応方法                                      |
| -------------- | ------------------- | --------------------------------------------- |
| 投影法         | JIS B 0001 第三角法 | 正面図の上に上面図、右に右側面図              |
| 投影法シンボル | ISO 128             | 図枠テンプレートに第三角法シンボルを配置      |
| 線種           | JIS Z 8312          | 外形=太線(0.35-0.5mm)、寸法=細線(0.18-0.25mm) |
| 寸法記入       | JIS B 0001          | 個数→寸法→加工法→深さ                         |
| 普通公差       | JIS B 0405-c        | 図枠注記「普通公差: JIS B 0405-c」            |
| R面取り        | JIS B 0001          | 「R6」、複数は「4-R3」                        |
| C面取り        | JIS B 0001          | 「C5」（45°×5mm）                             |
| 断面図         | JIS B 0001          | 太一点鎖線+矢印+アルファベット識別            |

## 10. システム全体像: 製造データパイプライン

本プロジェクトの最終形は「図面生成」だけではなく、**製造データパイプライン全体**の自動化である。

### 10.1 データフロー

```
Excel / CSV（入力）
    │
    ▼
製品データモデル（中間表現）
    │
    ├──→ 製造図面（PDF / DXF）         … 製造パイプライン
    │     └ 12シート × パーツ数（JIS B 0001 第三角法準拠）
    │
    ├──→ BOM（部品表）                  … 製造 + 資材管理の橋渡し
    │     ├ 部材リスト（MDF板: 品番・厚さ・幅×長さ・数量）
    │     ├ 金物リスト（木ダボ・ビス・ヒンジ・ラッチ: 品番・数量）
    │     └ 仕上材リスト（オレフィンシート: 品番・色・使用面積）
    │
    ├──→ 資材管理（MC）連携            … 資材パイプライン
    │     ├ 所要量計算（BOM × 生産数量 → 必要資材量）
    │     ├ 在庫引当（MDF板材・金物・仕上材の在庫照合）
    │     ├ 発注データ（不足資材の発注リスト）
    │     └ カットリスト（MDF板取り最適化 → 歩留まり管理）
    │
    └──→ 製造指示（将来拡張）
          ├ CNCボーリングデータ（穴位置・径・深さ）
          └ モルダープログラム（断面プロファイル指示）
```

### 10.3 設計原則

1. **製品データモデルを中心に据える**: 図面・BOM・資材管理データは全て同一モデルから派生
2. **パラメータ駆動**: Excel の 2-3 パラメータで全出力が連動更新
3. **2つのパイプライン**:
   - **製造パイプライン**: Excel → 製品データ → 図面（PDF/DXF）
   - **資材パイプライン**: Excel → 製品データ → BOM → 所要量計算 → 在庫引当 → 発注
4. **段階的な出力拡張**: Phase 2a=図面+BOM、Phase 2b=資材管理連携、Phase 3=製造指示
5. **データ形式の標準化**: BOM は CSV/JSON、資材管理データは既存MCシステムの入力形式に準拠
